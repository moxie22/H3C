01 — 页面目标

问题背景

寻源专员 / 采购员在「询价单」页面面对大量单据时，缺乏对询价池的全局视角，难以第一时间发现进度滞后的单据、供应商响应不足的风险，以及供应商 / 物料维度的集中度问题；逐单翻表效率低，缺少数据驱动的跟催与策略依据。

核心价值

在询价单页面直接嵌入 AI 分析能力，对询价单池做实时智能解读（状态分布、风险预警、趋势、供应商 / 物料维度）并支持自然语言查询，帮助采购员更快掌握询价健康度、识别风险、做出策略决策，无需离开当前页面。

成功指标

(1) AI 分析结论与人工经验判断方向一致率 ≥ 80%；(2) 分析响应时间 < 3 秒；(3) 用户无需离开询价单页面即可完成分析与查询。

02 — 用户故事

角色职责核心关注使用场景
寻源专员 / 采购员发起询价、跟进报价、推动比价核价哪些单子快逾期、谁不报价、整体健康度日常询价跟催、周度询价盘点、临时查询某单

用户故事：作为寻源专员，我希望在询价单页面一眼看到 AI 能帮什么（全局分析入口），并能用自然语言询问任意单据，以便快速掌握询价健康度、识别将要逾期的单据与响应不足的供应商，减少逐单翻表的时间成本，降低遗漏风险的可能性。

03 — 信息架构

入口位置

顶部 Header 栏右侧：「需求文档」白底按钮（左） + 「AI助手」青色渐变胶囊按钮（右）；左侧菜单含「AI助手」项（机器人图标）。

AI 入口（克制，不抢列表主角）

「询价单」页面主内容区是询价单表格（工作列表页，列表为主角）。AI 入口保持轻量：顶部 Header 右侧「AI助手」青色胶囊按钮 + 左侧菜单「AI助手」项，二者均唤起右侧 420px 抽屉对话。不在列表上方放置大块 AI 主卡，避免喧宾夺主。

右侧抽屉（对话区）

点击 Header / 左侧「AI助手」即滑出 420px 抽屉，含圆形头像、问候语、6 个自然语言型快捷场景（覆盖 SC-01~06）与底部输入框。抽屉采用纯遮罩（overlay, rgba(0,0,0,0.25)）覆盖模式，打开时主内容不位移。

询价单列表（主数据区）

AI 主卡下方为询价单表格：状态分组页签（全部/草稿/报价中/比价中/核价中/已完成/已取消）、关键字搜索、列表展示编号/标题/状态/截止/报价进度/被邀供应商数。

技术方案

AI 智能助手为独立前端原型，当前演示阶段将询价单数据以 Mock JSON 直接写入同一 HTML 中，由前端规则引擎完成分析与意图路由（未来可替换为 LLM API + 真实 ERP/MDM 数据源）。

核心功能特性（6 类策略场景）

  • SC-01 状态分布：6 状态计数 + 占比，整体健康度。
  • SC-02 风险预警：截止 ≤3 天且供应商报价进度 < 50% 的高优先跟催清单。
  • SC-03 供应商响应：平均响应率、低响应供应商提示。
  • SC-04 物料集中度：单一来源 / 高度集中物料识别。
  • SC-05 趋势与负荷：近 14 天新建 vs 未来截止双折线。
  • SC-06 对话式即时查询：状态 / 风险 / 单条详情 / 汇总 / 公开占比的自然语言问答。

04 — 策略分析业务场景

「询价单」的策略分析，目标是把分散的单据转化为可执行的采购策略信号。共梳理 6 个策略分析业务场景，覆盖「健康度 → 风险 → 供应商 → 物料 → 趋势 → 即时查询」全链路：

编号场景业务问题（采购员在想什么）支撑的决策
SC-01健康度总览
状态分布分析		当前询价单池整体是否健康？是否大量积压在草稿 / 已取消？询价 pipeline 节奏管理、需求澄清推进
SC-02风险预警
逾期与临近截止		哪些单子即将到期却还没收到足够报价？哪些已经逾期？跟催优先级分配、避免单据作废
SC-03供应商响应
行为分析		被邀请的供应商里，谁经常不报价？整体响应率如何？供应商池激活 / 淘汰、确保有效报价数
SC-04物料集中度
供应风险分析		哪些物料的询价高度集中在少数供应商？是否单一来源？引入备选供应商、分散供应风险
SC-05趋势与负荷

SC-01　询价健康度总览（状态分布分析）

  • 业务问题：询价单池整体是否健康？草稿积压、已取消占比是否过高？
  • 触发方式：抽屉快捷场景"询价单汇总分析"；对话"询价单整体状态分布怎样"。
  • 分析逻辑：按状态分组计数（草稿 / 报价中 / 比价中 / 核价中 / 已完成 / 已取消），计算各状态占比、草稿积压率、已取消率。
  • 输出：6 状态计数 + 占比 + 关键指标（平均被邀供应商数、公开询价占比）。
  • 关联决策：草稿占比高 → 推动业务及时完善并提交；已取消率高 → 排查供应商响应或需求澄清问题。

SC-02　逾期与临近截止风险预警

  • 业务问题：哪些询价单即将到期但还没收到足够报价？哪些已逾期？
  • 触发方式：抽屉快捷场景"风险预警清单"；对话"哪些询价单快截止、有风险"。
  • 分析逻辑：筛选「截止 ≤ 3 天 且 供应商报价进度 < 50% 且 状态非终态」的单据，按剩余天数升序。
  • 输出：风险清单表（编号 / 状态 / 截止 / 进度 / 剩余天数），剩余 ≤ 1 天标红高亮。
  • 关联决策：生成跟催优先级清单，分配给对应寻源专员催促供应商报价，避免单据过期作废。

SC-03　供应商响应行为分析

  • 业务问题：被邀请的供应商里，谁经常不报价？谁报价积极？整体响应率如何？
  • 触发方式：抽屉快捷场景"供应商响应分析"；对话"哪家供应商经常不报价"。
  • 分析逻辑：按供应商聚合 被邀次数、实际报价数、响应率 = 报价 / 被邀、平均报价水平、入选率。
  • 输出：供应商响应排行（高 / 低响应），低响应供应商警示。
  • 关联决策：对长期低响应供应商激活 / 淘汰；优化供应商池；对关键物料确保 ≥ N 家有效报价。

SC-04　物料集中度与供应风险分析

  • 业务问题：哪些物料的询价高度集中在少数供应商？是否存在单一来源风险？
  • 触发方式：抽屉快捷场景"物料集中度分析"；对话"哪些物料供应集中、单一来源"。
  • 分析逻辑：按物料聚合 询价次数、涉及供应商数、报价分布；识别「涉及供应商数 = 1」或「报价高度集中」的物料。
  • 输出：物料维度表（询价次数 / 供应商数 / 价格区间）+ 单一来源警示。
  • 关联决策：对单一来源物料推动引入备选供应商，分散供应风险。

SC-05　询价趋势与产能负荷分析

  • 业务问题：近期新建询价是否突增？未来哪几天是截止高峰？人力 / 产能是否瓶颈？
  • 触发方式：对话"近期询价趋势、哪天截止高峰"。
  • 分析逻辑：近 14 天每日新建量 vs 未来截止量 双折线对比。
  • 输出：双折线趋势图 + 峰值标注。
  • 关联决策：截止高峰前提前排程寻源人力；新建突增时排查批量需求或异常。

SC-06　AI 对话式即时查询

  • 业务问题：采购员想即时知道"报价中的有哪些""RFQxxx 详情""公开询价占比"而不翻表。
  • 触发方式：右侧 AI 抽屉，输入自然语言。
  • 分析逻辑：意图路由（状态查询 / 风险 / 单条详情 / 汇总 / 公开占比）→ 前端规则引擎检索 Mock 数据。
  • 输出：文字 + 表格回复，支持连续追问。
  • 关联决策：即时决策支持，减少页面跳转与人工筛选。

05 — 提示词、Skill与Tool设计

原型阶段采用「1 个全局调度 + 5 类查询意图」的轻量架构：全局调度负责意图识别与编排，各意图由前端规则引擎（Skill）执行，Tool 提供 Mock 数据检索。

▍ 全局调度 Prompt（意图路由器）

你是「立讯 AI 采购助手」的意图路由器。把用户一句话精确分类到唯一意图：状态查询、风险预警、单条详情、总量汇总、公开占比；非上述意图时引导用户用示例提问。不确定时不编造数据，明确说明"当前数据不足以判断"。

▍ 查询意图（Skill）

  • 状态查询 Skill：识别状态关键词（草稿/报价中/比价中/核价中/已完成/已取消），返回该状态单据清单与计数。
  • 风险预警 Skill：计算截止剩余天数与报价进度，返回高风险清单。
  • 单条详情 Skill：解析 RFQ 编号，返回该单据全字段。
  • 总量汇总 Skill：返回总量、状态分布、平均被邀供应商数、公开占比。
  • 公开占比 Skill：返回公开 / 非公开询价占比。

▍ Tool（原子能力）

  • query_inquiries：按状态 / 关键词 / 编号检索询价单 Mock 数据。
  • calc_risk：基于截止时间与报价进度计算风险等级。
  • aggregate：按状态 / 供应商 / 物料维度聚合统计。

06 — 状态规则

询价单状态定义

状态含义是否终态
草稿单据创建但未提交 / 未正式发起否
报价中已发给供应商，等待报价否
比价中已收齐报价，进行比价否
核价中比价完成，进入核价审批否
已完成核价通过，流程结束是
已取消单据作废是

风险判定阈值

  • 高风险：剩余天数 ≤ 3 且 供应商报价进度 < 50%（且状态非终态）。
  • 剩余天数 = 截止时间 − 当前日期；≤ 1 天额外标红。
  • 响应率 = 实际报价供应商数 / 被邀供应商数；< 50% 视为低响应警示。

07 — 异常处理

  • 未检测到询价单数据 → 提示"当前没有询价单数据"。
  • 某状态下无单据 → 该状态卡显示 0，不报错。
  • 风险清单为空 → 显示"当前无高风险询价单"。
  • 对话无法识别意图 → 返回可用问法引导，不编造数据。
  • 自然语言查不到编号 → 提示未找到，并给出示例格式。
