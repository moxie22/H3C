01 — 页面目标

问题背景

采购员创建物料时需手工填写大量属性字段（物料名称、分类、规格参数等），耗时长且易出错。尤其面对非标物料或仅有图片/规格书的场景，属性录入效率极低。

核心价值

在物料创建页面嵌入AI识别能力，采购员上传物料图片或规格书后，AI自动识别物料属性并填充表单，将物料创建时间从分钟级降至秒级。

成功指标

(1) AI识别准确率 >= 85%（核心属性：名称、分类、规格参数）
(2) 单次识别响应时间 < 5秒（超时阈值10秒，超过则降级为手动填写）
(3) 物料创建平均耗时降低 >= 60%
   · 基线：当前手工创建一个物料平均耗时约8分钟（含查规格书、填字段、校对）
   · 目标：AI辅助后平均耗时降至3.2分钟以内（识别+核对+补充+提交）

02 — 用户故事

角色职责核心关注使用场景
采购员创建物料、维护物料属性填表效率、属性准确性日常新建物料，批量录入
物料管理员审核物料、维护分类体系分类合理性、属性完整性审核新建物料，补充属性
用户故事：作为采购员，在需要创建新物料时，我希望能上传物料图片或规格书，由AI自动识别物料属性并填充创建表单，获得物料名称、分类、规格参数等关键属性的智能推荐，以便在无需逐字段手工录入的情况下，快速完成物料创建，减少录入错误和时间成本。
用户故事（物料管理员）：作为物料管理员，在审核采购员提交的新建物料时，我希望能查看AI识别的原文来源和识别依据（识别澄清），快速判断属性是否准确；对于置信度低的字段能够直接通过对话修改或手动调整，以便高效完成物料审核和分类体系维护，确保物料库数据的准确性和完整性。

03 — 信息架构

页面布局（V2.0）

采用左右分屏布局：左侧为AI对话区（含对话历史、需求下单/查询库存快捷入口），右侧为物料创建页面。右侧内部分为左右两列，同等高度：

(1) 左列：AI识别组件（固定宽度~340px）—— 上下两部分组成一个完整组件：
   · 上方：上传控件：物料名称输入框 + 图片/文档上传区（含更换图片、页码）+ 开始识别按钮 + 识别成功提示
   · 下方：识别队列：处理进度条 + 状态色点（已完成/处理中/待处理）+ Tab筛选（全部/已完成/已保存/处理中/待处理）+ 队列项列表
(2) 右列：物料信息模块（弹性宽度）—— Tab切换（物料信息 / 相识物料 / 重置物料）+ 基本信息（100×100原图缩略图 + 9字段网格）+ 备注 + 开关组 + 物料属性（2列8项）+ 底部操作栏

AI对话区能力（V2.0）

· 识别澄清：识别完成后，对话区自动推送一条澄清消息，以表格形式展示每个字段的「识别值 / 原文 / 识别依据」，让采购员了解每个属性的数据来源
· 对话改字段：采购员可在对话框输入指令（如「物料名称改成电磁阀」「计量单位改成套」），AI解析后实时更新右侧表单对应字段并高亮标注，同时回复确认消息
· 扩充别名匹配 V2.0：支持20+组字段别名（如「交期」→订货周期、「牌子」→品牌、「层次」→产品层次等），降低用户记忆精确字段名的成本
· 保存校验反馈 V2.0：点击保存时若有必填字段缺失，对话区自动推送缺失字段列表，引导采购员补充
· 多物料选择 V2.0：图片中检测到多个物料时，AI列出候选物料列表供采购员选择主物料

表单字段分组

· 基本信息：物料图片、物料名称、物料组、品牌、物料类型、物料单位、产品层次、环保标记、订货周期、旧料号
· 规格参数（以气缸为例）：气缸接管口径、气缸安装方式、气缸缓冲形式、气缸动作形式、气缸缸径、气缸行程
· 补充信息：产地、材质

顶部入口

04 — 业务流程

完整物料创建流水线（V2.0）：

步骤动作详情
1. 触发打开物料创建页面页面默认展开AI识别组件，直接在右分屏操作
2. 输入输入物料名称 / 上传图片在AI识别组件上方上传区输入物料名称，或上传物料图片/PDF/Word/Excel（支持Ctrl+V粘贴或拖拽）
3. 识别点击「开始识别」AI调用识别引擎，从图片/文档中提取物料属性，同时根据物料名称匹配分类体系
4. 填充自动填充表单识别成功后，右侧物料信息模块自动填充物料名称、分类、规格参数等字段，识别组件下方队列显示处理状态
4a. 多物料选择 V2.0检测到多个物料时选择主物料若图片中检测到多个物料，AI提示「检测到多个物体」，采购员从列表中选择主物料，仅将主物料属性填充表单
5. 澄清对话区自动推送识别澄清AI在对话区推送一条澄清消息，以表格展示每个字段的「识别值 / 原文 / 识别依据」，让采购员了解数据来源和推理过程
6. 修改对话改字段 / 手动修改采购员可在对话区输入指令（如「物料名称改成电磁阀」）由AI实时更新表单字段并高亮；也可直接在表单中手动修改
7. 校验自动校验 + 人工核对系统自动调用 validate_material_skill 校验必填字段完整性，标注缺失项；采购员核对所有属性，补充缺失字段
8. 提交保存物料点击底部「确认保存」按钮，系统再次校验必填字段（未通过则高亮提示+对话区推送缺失列表），通过后提交物料创建请求并完成创建
流程图：打开页面 → 输入/上传 → 开始识别 → AI提取属性 → [多物料?] → 选择主物料 → 自动填表 → 对话澄清（原文+依据）→ 对话改字段/手动修改 → 自动校验+人工核对 → 保存提交
异常分支：识别失败 → 提示错误原因 → 手动填表 → 保存提交
对话改字段分支：用户输入指令 → AI解析字段名+新值 → 匹配表单字段 → 更新值+高亮 → 回复确认消息
多物料分支：检测到多个物体 → 列出候选物料 → 采购员选择主物料 → 仅填充主物料属性

05 — 提示词、Skill与Tool设计

本章定义物料创建Agent的完整能力体系，由三层构成：Prompt（提示词）驱动LLM的识别逻辑，Skill（技能）封装可复用的识别能力，Tool（工具）提供数据获取与系统交互的原子操作。

▍ 全局调度 Prompt

你是一个专业的物料创建调度助手，服务于供应链采购团队的物料创建场景。 ## 角色定位 你是物料创建的「调度中枢」——接收采购员的物料创建请求，判断输入类型（文本/图片/文档），调度识别和匹配能力，汇总结果并自动填充表单。 ## 调度规则 1. 接收物料名称或上传文件 → 调用 material_recognition_skill 2. 识别结果 → 调用 classify_material_skill 匹配分类 3. 若检测到多个物料 → 列出候选列表，等待采购员选择主物料 4. 分类结果 → 调用 fill_form_skill 自动填充表单 5. 填充完成 → 调用 clarify_recognition_skill 推送识别澄清（原文+依据） 6. 采购员通过对话修改字段 → 调用 chat_to_form_skill 解析指令并更新表单 7. 提交前 → 调用 validate_material_skill 校验必填字段完整性 8. 校验通过 → 调用 save_material 提交；未通过 → 标注缺失项，引导补充 ## 行为准则 - 不确定时不编造属性，明确标注"未识别到，请手动填写" - 物料名称必须准确，不可臆测 - 分类匹配置信度 < 70% 时标注"建议人工确认" - 识别澄清须如实标注原文来源和推理依据，不可编造 - 对话改字段时须高亮标注修改的字段，并回复确认消息 - 语气专业、高效，减少采购员等待焦虑

▍ 维度一：物料识别 Prompt

你是一个专业的物料识别专家，从图片或规格书中提取物料核心属性。 ## 输入数据 - 图片（物料实物照片 / 规格书截图） - 或文档（PDF / Word / Excel 格式规格书） - 或物料名称文本 ## 识别规则 - 从图片中识别物料外观特征 → 推断物料类型 - 从规格书中提取：物料名称、型号、规格参数、材质、品牌 - 从物料名称文本中提取关键词 → 匹配物料知识库 ## 输出格式 { "materialName": "气缸", "materialType": "气动元件", "specifications": { "interfaceDiameter": "M5", "mountingMethod": "脚座安装", "bufferType": "气缓冲", "actionType": "单作用" }, "unit": "个", "confidence": 0.92 }
