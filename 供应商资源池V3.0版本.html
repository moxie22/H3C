01 — 页面目标

问题背景

供应商资源池中沉淀着大量「未引入 / 引入中」状态的供应商。采购寻源人员在需要引荐 / 引入新供应商时，缺乏统一、可量化的评估口径，往往凭经验或主观印象挑选——效率低、容易漏掉高匹配度供应商，也难以向决策层解释「为什么引荐这一家」。

核心价值

在供应商资源池页面直接嵌入 AI 引荐能力，对齐业务侧《新供方现场审核项目方案 V6.0》的评估全集：以「质量 45% + 商务 45% + 技术 10%」三维加权为总评口径。把可计算的资质与档案数据自动算分，把必须到厂核实的项（现场体系执行、工艺设备、5S、社会责任、研发样件等）统一交给「稽核项目考察」由人工确认，避免 AI 编造现场结论。最终给出可解释、可复核、可现场回填的引荐结论。

成功指标

1）AI 引荐建议与引入评审员经验判断方向一致率 ≥ 80%；
2）整体分析响应时间 < 3 秒；
3）用户无需离开资源池页面即可完成引荐评估与方案提交。

03 — 信息架构

入口位置

资源池顶部工具栏「⚡ 整体引入分析」按钮；每行操作列「AI 分析」按钮；右侧抽屉「AI 助手」；顶部「需求文档」按钮。

四种触发方式

1）整体引入分析——工具栏按钮，对全库候选做三维加权引荐；
2）行内 AI 分析——数据行「AI 分析」按钮，打开面板并自动对该行供应商深度分析；
3）对话 Intake——在 AI 助手对话框输入自然语言（如「帮我推荐气缸的供应商」），前端关键词引擎前缀匹配自动定位级联路径并触发分析，无需调用 LLM；
4）需求文档——顶部按钮弹出本模态框。

面板形态

右侧抽屉滑出（宽度适配窄栏），采用纯遮罩覆盖模式，打开时主内容区域不发生位移。

技术方案

当前前端原型为演示简化，将资源池数据直接写入同一 HTML。真实部署时，AI 助手作为独立系统通过 <iframe> 嵌入采购平台，两系统通过 window.postMessage 双向通信：平台推送候选数据 → AI 执行规则引擎分析 → 用户确认引荐方案后 postMessage 回传（方案 ID / 名称 / 供应商 / 策略）→ 平台触发下游引入审批流。

核心功能特性

1）四大业务模块：引荐供应商推荐池 / 引荐分析（先展示「根据当前集团询价单 / 采购订单推导分析」，再展示需求画像对比表与供应商画像对比）/ 稽核项目考察 / 引荐方案，以可折叠卡片呈现。

2）整体与明细行统一：两者复用同一套分析卡片，差异仅在是否已选定供应商，行内分析不出现物料组选择与「引荐决策」卡片。

3）对话 Intake 与三级级联：抽屉内对话卡片含三级联动级联（一级→二级→三级类目）选目标品类，未选 = 全品类引荐。

4）素质雷达图：六维（质量保障 / 商务信用 / 技术能力 / 交付能力 / 风险合规 / 财务健康）纯前端 SVG，无外部图表库（仅用于推荐池展开画像，引荐分析对比表不再含雷达图）。

5）稽核判断依据：每项给出 AI 初判与判断依据，卡片内「稽核对象」下拉可切换候选，人工评价可逐项修改。

6）引荐方案：列出优选 / 备选两方案，选择后二次确认，确认即调用引荐接口（桩）回传下游；另支持批量引荐（多供应商批量引入）。

04 — 业务流程

阶段步骤说明
1打开助手进入 AI 抽屉，对话流出现「目标品类级联」卡片
2候选获取拉取资源池「未引入 / 引入中」候选集合
3目标品类确定三级级联选择物料组 → 取其路径集；未选 → 全品类引荐（路径集为空，品类相关性计满分）
4触发分析整体分析 / 行内分析 / 对话 Intake（关键词命中自动定位级联路径）
5三维加权评分资源池字段 + 供应商档案(模拟) 全可计算 → 质量×45% + 商务×45% + 技术×10%
6输出推荐池按综合分筛选 1–5 家优选供应商，紧凑清单、默认折叠，可展开素质雷达图与基础画像
7画像对比需求画像 vs 选中供应商素质雷达图对比，定位差距
8引荐分析先展示「根据当前集团 26年Q2 询价单 / 采购订单推导分析」：26年Q2 季度 PO / RFQ 订单量、物料需求数量与环比（按季度统计），按紧急程度排序（紧急·缺口 > 偏紧·关注 > 平稳 > 放缓）并标出缺口；再展示需求画像（库内基准）对比表与供应商画像对比；按供应商可供物料组匹配系统整体需求，需求激增且可服务供应商少 → 急需引荐，无对应采购需求 → 暂不推荐
9稽核考察逐项确认稽核点（含 AI 判断依据与人工评价下拉），形成稽核结论
10引荐方案在「引荐方案」中选择优选 / 备选（单家）或批量引荐（多供应商）；批量引荐勾选供应商（可几十家）并选引入策略，二次确认后调用引荐接口触发下游引入审批流

05 — 提示词、Skill与Tool设计

本章定义引荐 Agent 的能力体系，由三层构成：Prompt（提示词）驱动 LLM 分析逻辑，Skill（技能）封装可复用分析能力，Tool（工具）提供数据获取与系统交互的原子操作。三者关系：Agent 接收请求 → 调用 Tool 获取数据 → 调度 Skill 执行分析（Skill 内部用 Prompt 驱动 LLM）→ 汇总渲染输出。

能力体系（三层）：Prompt 驱动 LLM 分析逻辑；Skill 封装可复用分析能力（质量分 / 商务分 / 技术分 / 引入状态 / 数据完整度）；Tool 提供原子操作——get_supplier_pool（候选集合）/ get_supplier_archive（供应商档案）/ get_demand_orders（按物料类目聚合的询价单 / 采购订单）/ render_analysis_cards（渲染四张卡片）/ call_introduce_api（提交引荐方案）。真实环境已对接，demo 中以模拟数据 / 桩实现。

提示词一：总提示词（入口调度 Agent）

00_master_prompt.md

# 供应商引荐 Agent · 总调度提示词

## 角色 你是供应商资源池的 AI 引荐「调度中枢」。接收采购寻源人员的分析请求，统筹质量 / 商务 / 技术 / 引入状态 / 数据完整度五条分析线，产出可解释、可复核、可现场回填的引荐结论。

## 工作流

  • 1）识别意图：整体引荐 / 行内分析 / 对话 Intake（自然语言 → 关键词命中级联路径）
  • 2）调用 Tool 取数 → 调度 Skill 评分 → 渲染四张卡片
  • 3）输出顺序固定：引荐供应商推荐池 → 引荐分析 → 稽核项目考察 → 引荐方案

## 输出纪律

  • 不编造现场结论：需到厂核实项一律交「稽核项目考察」人工确认
  • 评分唯一口径：综合分 = 质量×45% + 商务×45% + 技术×10%
  • 推荐池只向用户展示单一「评分」数值，不拆分「引荐评分 / 综合评分」

提示词二：引荐供应商推荐池提示词

01_recommend_pool_prompt.md

# 引荐供应商推荐池提示词

## 目标 从「未引入 / 引入中」候选集合中筛出 1–5 家最值得引荐的供应商，形成紧凑推荐池。

## 输入 候选集合（资源池字段 + 供应商档案）；目标品类路径 targetPaths（空 = 全品类引荐）。

## 计算

  • 三维加权：质量×45% + 商务×45% + 技术×10%，每项 0–100
  • 品类相关性 categoryRel：命中 targetPaths 任一级节点 = 1；非目标 = 0.4；无标签 = 0.55~1.0 折扣；全品类 = 1
  • 综合分 = 三维加权分 × categoryRel

## 排序与分级

  • 先按「需求引荐等级」：急需引荐 > 可引荐 > 暂缓引荐 > 暂无对应采购需求
  • 同等级内按综合分降序；取前 1–5 家
  • 若全部 < 65，取最高 1 家兜底

## 输出字段（每行） 名称 / 资源编号 / 服务品类 / 评分（单一数值）/ 可展开：素质雷达图 + 基础画像。

提示词三：引荐分析提示词

02_intro_analysis_prompt.md

# 引荐分析提示词

## 目标 解释「为什么引荐 / 为什么暂缓」，以当前集团采购需求驱动。

## 第一部分：需求推导分析（标题固定为「26年Q2季度物料需求匹配」）

  • 数据源：26年Q2 询价单 / 采购订单（统计周期 = 4–6 月，按季度统计）
  • 范围：取该供应商 service 命中物料组，过滤系统 26年Q2 需求，最多展示 3 个物料组
  • 每行双指标：PO 采购订单（季度订单量 / 季度物料需求 / 环比 vs 26年Q1）、RFQ 询价单（季度询价量 / 季度物料需求 / 环比）
  • 紧急程度 4 级：PO 环比 ≥ 20% 且可服务供应商 ≤ 3 → 紧急·缺口（红）；≥ 15% 且 ≤ 4 → 偏紧·关注（橙）；< 0 → 放缓（黄）；其余 → 平稳（绿）
  • 卡片底部左对齐：「可供货当前物料组的供应商数 N 家」（N 高亮加粗）
  • 无对应采购需求 → 空态「26年Q2 季度暂无对应集团采购需求，暂不纳入优先引荐」

## 第二部分：画像对比

  • 需求画像（库内基准）vs 选中供应商画像逐项对比，含差距列
  • 基础画像字段：注册资本 / 成立年限 / 实缴资本 / 社保人数 / 企业规模 / 企业地区
  • 本模块不含素质雷达图

提示词四：稽核项目考察提示词

03_audit_prompt.md

# 稽核项目考察提示词

## 目标 把必须到厂核实的项交由人工确认；AI 仅给初判与依据，绝不远程判定现场结论。

## 维度分组（11 项） 商务资质 / 商务风险 / 财务健康 / 履约能力 / 质量体系 / 质量绩效 / 变更管理 / 现场管理 / 社会责任 / 技术能力 / 工艺设备。

## 每项输出

  • AI 初判徽标（满足 / 不满足 / 待核）
  • 判断依据（关键条款 ★ 标记）
  • 人工评价：满足 / 不满足 / 待核（分段单选，逐项确认）

## 汇总

  • 顶部「稽核结论」条：通过 / 待现场复核 / 不通过，含按维度通过率迷你条
  • 现场项未确认 → 通过率仅计已评项，提示「含未现场核实项」
  • 社会责任（童工 / 强迫劳动 / 冲突矿物）AI 无法远程判定，仅列待核

提示词五：引荐方案提示词

04_plan_prompt.md

# 引荐方案提示词

## 目标 基于推荐池与稽核结论，给出可提交下游的引荐方案。

## 三种引荐方式

  • 1）优选方案：标准合格引入（单家）
  • 2）备选方案：带条件复核的引入（单家）
  • 3）批量引荐：多供应商批量引入（支持几十家，源自全量「未引入」候选）

## 单家流程 选择优选 / 备选 → 二次确认卡片展示方案 ID / 名称 / 供应商 / 策略 → 确认即调用 call_introduce_api（桩）postMessage 回传 → 触发下游引入审批流。

## 批量流程

  • 勾选供应商（全选 / 清空 / 已选计数），选引入策略（标准合格引入 / 试单观察名录）
  • 勾选 ≥ 1 家时确认按钮可用；二次确认截断至前 10 家 + 「等共 N 家」
  • 确认即逐一调用 call_introduce_api，回显「已批量提交 N 家」

## 纪律 引荐方案优先取推荐池排序靠前的供应商；提交失败保留方案选择状态，提示「引荐提交失败，请重试」。

附：三维加权子项与权重

## 分析规则 - targetPaths 为空 → 全品类引荐，categoryRel = 1（所有候选满足） - targetPaths 非空 → 候选 service 命中任一路径节点（一/二/三级）即 categoryRel = 1，其它品类 = 0.4，无标签 = 0.55~1.0 折扣 - 需现场核实的子项不参与数值评分，统一在「稽核项目考察」中确认 商务 45%：注册资本(0.20) · 经营年限(0.14) · 社保人数(0.12) · 诉讼/处罚风险(0.14) · 财务健康/负债率(0.12) · 交期准时率(0.16) · 客诉率(0.10) · ISO9001(0.02) 质量 45%：ISO9001(0.30) · 客诉率(0.30) · 变更控制/PCN(0.20) · 交期准时率(0.20) 技术 10%：设计变更/ECN/NPI(0.60) · 资料完整度(0.20) · 交期准时率(0.20) 综合分 = 质量×45% + 商务×45% + 技术×10%；每项 0–100；维度分 = 子项加权平均 推荐池「评分」= 综合分（单一数值）；排序 = 先需求引荐等级，同等级内按评分降序

V6.0 四表（汇总 + 质量 + 商务 + 技术，约 155 审查项）为业务真实评估全集，本助手结构对齐：质量体系评审 → 质量 45%、商务体系评审 → 商务 45%、技术体系评审 → 技术 10%、汇总表 → 三维加权总评；多数现场项由「稽核项目考察」人工确认，可算项计入数值分。

06 — 状态规则

状态触发条件界面表现

分析中点击分析按钮后按钮 loading 态；骨架屏 + 逐条输出动画

面板关闭用户关闭抽屉分析结果保留（重新打开仍可见上次结论）

07 — 异常处理

异常场景处理方式
资源池无候选数据提示「当前资源池未检测到可引荐候选」，不执行分析
仅 1 家候选推荐池仅 1 家；引荐分析无对照对象，标注「单一候选」
全品类模式（未选物料组）categoryRel = 1，所有候选参与引荐，不弹物料组选择
物料组无匹配候选提示「该品类下未检测到达标候选，建议放宽条件或走全品类」
现场项未确认即提交稽核「需现场」项未填，通过率仅计已评分项，提示「含未现场核实项」

Intake 意图模糊用户输入非分析意图（如「你好」），走普通对话模式，不触发分析
数据动态刷新后提示「页面数据已变更，建议重新分析」；上次分析标记过期

AC-04引荐分析含两部分：①先展示「根据当前集团 26年Q2 询价单 / 采购订单推导分析」——需求概览（标题：26年Q2季度物料需求匹配）按「当前对比供应商的可供物料组」自适应：取该供应商可服务的物料组（真实环境取 service 命中项，演示按 index 映射 2 个类目），过滤系统 26年Q2 季度采购需求到该范围，按紧急程度排序最多展示 3 个物料组。统计口径：订单量与物料需求数量均以季度为统计周期（26年Q2 = 4–6 月），表头标注「季度订单量 / 季度物料需求」；环比 = 本季度相对上一季度（26年Q1）的变化率。每个物料展示两行（PO 采购订单：季度订单量 / 季度物料需求数量 / 环比；RFQ 询价单：季度询价量 / 季度物料需求数量 / 环比）。紧急程度分级规则（4 级）：PO 环比 ≥ 20% 且 可供货供应商 ≤ 3 → 紧急·缺口（红，卡片高亮）；PO 环比 ≥ 15% 且 可供货供应商 ≤ 4 → 偏紧·关注（橙）；PO 环比 < 0 → 放缓（黄）；其余 → 平稳（绿）。卡片底部统一左对齐展示「可供货当前物料组的供应商数 N 家」，其中数字 N 高亮加粗。若该供应商无对应采购需求则展示空态提示「26年Q2 季度暂无对应集团采购需求，暂不纳入优先引荐」；切换对比供应商时，需求概览与对比表一并重算重渲染，不会出现分析丢失。推荐池需求匹配按供应商可供物料组 × 系统整体需求给出引荐策略；②需求画像（库内基准）对比表与选中供应商画像逐项对比（差距列），对比对象默认排序首位、可切换；基础画像字段为注册资本 / 成立年限 / 实缴资本 / 社保人数 / 企业规模 / 企业地区（不含交期准时率与服务品类）；该模块不再含素质雷达图。版式一致性：「需求概览」「推荐池需求匹配」「需求画像对比」三个子板块统一采用带标题圆点的独立分区卡片（.sub-section），标题/小标题/文本字号与配色一致，子板块之间有明显边框分区。
AC-05稽核项目考察卡片内「稽核对象」下拉（不展示于标题）；顶部「稽核结论」汇总条（通过 / 待现场复核 / 不通过，按通过率与待核项判定）含按维度（商务资质 / 商务风险 / 财务健康 / 履约能力 / 质量体系 / 质量绩效 / 变更管理 / 现场管理 / 社会责任 / 技术能力 / 工艺设备）的通过率迷你条；各维度以分组卡片呈现，每项含 AI 初判徽标、判断依据、关键条款★标记，人工以「满足 / 不满足 / 待核」分段单选（替代原下拉）逐项确认

AC-08引荐方案提供三种引荐方式：①优选方案（标准合格引入，单家）；②备选方案（带条件复核的引入，单家）；③批量引荐（多供应商批量引入，支持几十家）。单家方案选择后二次确认即调用引荐接口；批量引荐需先勾选供应商（全量「未引入」候选，支持几十家）、选定引入策略（标准合格引入 / 试单观察名录），确认后逐一调用引荐接口触发下游业务流

黄金集（Golden Set）评测用例

以下 5 个用例为测试人员评测时的标准输入 / 预期输出基准，覆盖正常、边界、极限、异常四类场景与评分 / 推荐池 / 画像 / 稽核 / 方案五环节；全部为必过项，任一不通过即视为版本不合格。

正常G1 · 全品类整体引荐覆盖：评分 / 推荐池 / 画像 / 稽核 / 方案　·　难度 Medium　·　必过

输入

点击工具栏「⚡ 整体引入分析」，不选择物料组（= 全品类引荐），等待分析完成。

预期输出

① 推荐池输出 1–5 家优选，每行仅含单一「评分」数值（无「引荐评分 / 综合评分」双值）；② 引荐分析展示「26年Q2季度物料需求匹配」需求概览与画像对比；③ 稽核 / 引荐方案卡片正常渲染。

通过标准

综合分 = 质量×45% + 商务×45% + 技术×10%；推荐池仅展示单一「评分」；排序先按需求引荐等级、同等级内按评分降序；四维卡片齐全。

正常G2 · 物料组级联引荐 · 缺口识别覆盖：评分 / 推荐池 / 画像 / 引荐分析　·　难度 Medium　·　必过

输入

在 AI 助手对话框输入「帮我推荐气缸的供应商」→ 关键词引擎命中三级级联路径 → 触发分析。

预期输出

① 推荐池仅含 service 命中「气缸」的候选；② 引荐分析需求概览出现「紧急·缺口」（气缸 PO 环比 ≥ 20% 且可服务供应商 ≤ 3），卡片红色高亮；③ 卡片底部左对齐「可供货当前物料组的供应商数 N 家」，N 高亮加粗。

通过标准

紧急程度 4 级规则成立（紧急·缺口 / 偏紧·关注 / 平稳 / 放缓）；供应商数 N 正确高亮；非命中品类候选不进入推荐池。

边界G3 · 综合分均 < 65 兜底取值覆盖：评分 / 推荐池　·　难度 Medium　·　必过

输入

构造候选综合分全部 < 65 的全品类场景（或直接选取低分候选集），触发整体分析。

预期输出

推荐池不为空，取综合分最高 1 家作为兜底输出，并标注其评分偏低。

通过标准

不返回空推荐池；兜底严格取综合分最高者（唯一 1 家）；评分计算口径正确。

极限G4 · 物料组无匹配候选覆盖：异常处理 / 推荐池　·　难度 Hard　·　必过

输入

选择一个无达标候选的冷门物料组（如某未覆盖三级类目）作为目标品类，触发分析。

预期输出

提示「该品类下未检测到达标候选，建议放宽条件或走全品类」；不展示伪推荐池；提供全品类退路入口。

通过标准

异常提示文案准确；推荐池为空而非错误数据；可一键切换全品类重分析。

异常G5 · 提交失败与重试覆盖：引荐方案 / 异常处理　·　难度 Hard　·　必过

输入

引荐方案选择「优选方案」→ 二次确认 → 模拟 call_introduce_api 返回失败（接口异常 / 超时）。

预期输出

① 提示「引荐提交失败，请重试」，保留已选方案与供应商状态；② 重试成功后回显「已提交」并触发下游；③ 超时（> 5 秒）显示「分析超时，请重试」并保留已完成维度。

通过标准

失败不改写已选方案；重试成功即回传下游；现场项未确认即提交时提示「含未现场核实项」。
